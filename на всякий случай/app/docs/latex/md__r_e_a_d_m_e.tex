{\bfseries{Название:\+}} RestN ~\newline
 {\bfseries{Тип:\+}} кроссплатформенное приложение для планшетов (Android /\+ Windows /\+ Linux) на Qt 6 ~\newline
 {\bfseries{Назначение:\+}} автоматизация приёма заказов, управления складом и формирования отчётности в ресторане.

{\bfseries{Ключевые возможности:\+}}


\begin{DoxyItemize}
\item Авторизация сотрудников (официант, кладовщик, администратор)
\item Оформление заказов с автоматическим списанием ингредиентов
\item Возврат продуктов при удалении позиции из заказа
\item Управление складскими остатками (добавление, убавка, установка)
\item Отчёты по выручке, популярности блюд и дефицитным продуктам
\item Сохранение отчётов в CSV
\item Ролевое разграничение доступа
\item Синхронизация данных по локальной сети (клиент-\/сервер)
\end{DoxyItemize}

\hypertarget{md__r_e_a_d_m_e_autotoc_md7}{}\doxysection{\texorpdfstring{Описание программных модулей}{Описание программных модулей}}\label{md__r_e_a_d_m_e_autotoc_md7}
\hypertarget{md__r_e_a_d_m_e_autotoc_md8}{}\doxysubsection{\texorpdfstring{Модуль работы с базой данных (DBWorker)}{Модуль работы с базой данных (DBWorker)}}\label{md__r_e_a_d_m_e_autotoc_md8}
Класс {\ttfamily \doxylink{class_d_b_worker}{DBWorker}} предназначен для выполнения всех операций с базой данных Postgre\+SQL. Открывает соединение в конструкторе, поддерживает переподключение через {\ttfamily reconnect}. Основные методы:\+


\begin{DoxyItemize}
\item {\ttfamily check\+User(login, password)} – аутентификация с хешированием SHA3-\/256 + соль.
\item {\ttfamily get\+Foods()} – возвращает список блюд с доступностью (рассчитывается по остаткам) и составом.
\item {\ttfamily get\+Products()} – список продуктов с остатками и минимальным порогом.
\item {\ttfamily update\+Product\+Quantity()} /\+ {\ttfamily add\+Product\+Quantity()} – изменение остатков.
\item {\ttfamily check\+And\+Consume(food\+Id, quantity)} – в транзакции проверяет наличие ингредиентов и списывает их.
\item {\ttfamily return\+Ingredients(food\+Id, quantity)} – возвращает ингредиенты на склад.
\item {\ttfamily create\+Order()} /\+ {\ttfamily add\+Order\+Item()} – создание заказа.
\item Методы отчётов:\+ {\ttfamily get\+Report\+Revenue}, {\ttfamily get\+Revenue\+By\+Day}, {\ttfamily get\+Food\+Popularity}, {\ttfamily get\+Report\+Low\+Stock}.
\item {\ttfamily save\+Report\+To\+CSV()} – сохранение отчёта в CSV.
\end{DoxyItemize}

Все публичные методы обрабатывают исключения ({\ttfamily \doxylink{class_database_ex}{Database\+Ex}}, {\ttfamily \doxylink{class_validate_ex}{Validate\+Ex}}), логируют ошибки и испускают сигнал {\ttfamily error\+Occurred} для QML.\hypertarget{md__r_e_a_d_m_e_autotoc_md9}{}\doxysubsection{\texorpdfstring{Модуль сетевого клиента (Tcp\+Client)}{Модуль сетевого клиента (Tcp\+Client)}}\label{md__r_e_a_d_m_e_autotoc_md9}
Класс {\ttfamily \doxylink{class_tcp_client}{Tcp\+Client}} дублирует интерфейс {\ttfamily \doxylink{class_d_b_worker}{DBWorker}}, но вместо прямых запросов к БД отправляет JSON-\/команды на сервер. Соединение устанавливается методом {\ttfamily connect\+To\+Server(host, port)}. Протокол обмена:\+ запрос/\+ответ в виде строк JSON, завершённых символом {\ttfamily \textbackslash{}n}. Для синхронного ожидания используется {\ttfamily wait\+For\+Response} с таймаутом 60 секунд. При потере соединения генерируется сигнал {\ttfamily error\+Occurred}.

Пример запроса:\+

\DoxyCodeLine{\{"{}command"{}:\ "{}checkUser"{},\ "{}requestId"{}:\ 1,\ "{}data"{}:\ \{"{}login"{}:\ "{}waiter1"{},\ "{}password"{}:\ "{}123"{}\}\}}

\end{DoxyCode}


Ответ сервера:\+ 
\begin{DoxyCode}{0}
\DoxyCodeLine{\{"{}response"{}:\ true,\ "{}requestId"{}:\ 1,\ "{}data"{}:\ \{"{}role"{}:\ "{}waiter"{},\ "{}name"{}:\ "{}Иванов"{},\ "{}userId"{}:\ 5\}\}}

\end{DoxyCode}
\hypertarget{md__r_e_a_d_m_e_autotoc_md10}{}\doxysubsection{\texorpdfstring{Модуль сетевого сервера (Tcp\+Server + Client\+Connection)}{Модуль сетевого сервера (Tcp\+Server + Client\+Connection)}}\label{md__r_e_a_d_m_e_autotoc_md10}
{\ttfamily \doxylink{class_tcp_server}{Tcp\+Server}} запускает серверный сокет и при подключении клиента создаёт объект {\ttfamily \doxylink{class_client_connection}{Client\+Connection}}. В {\ttfamily Client\+Connection::\+on\+Ready\+Read()} данные накапливаются в буфере, извлекаются по разделителю {\ttfamily \textbackslash{}n}, парсится JSON и вызывается соответствующий метод {\ttfamily \doxylink{class_d_b_worker}{DBWorker}}. Результат отправляется обратно в виде JSON. Сервер способен параллельно обрабатывать несколько клиентов.\hypertarget{md__r_e_a_d_m_e_autotoc_md11}{}\doxysubsection{\texorpdfstring{Модуль логирования (Logs)}{Модуль логирования (Logs)}}\label{md__r_e_a_d_m_e_autotoc_md11}
Класс {\ttfamily \doxylink{class_logs}{Logs}} записывает сообщения с уровнями {\ttfamily INF}, {\ttfamily WARN}, {\ttfamily ERR} в файл и дублирует их в консоль. При недоступности основного файла использует резервный путь в системной папке приложения. Формат строки:\+ {\ttfamily yyyy-\/\+MM-\/dd hh:\+mm:\+ss \+[УРОВЕНЬ]\+ сообщение}.\hypertarget{md__r_e_a_d_m_e_autotoc_md12}{}\doxysubsection{\texorpdfstring{Модуль настроек (Settings)}{Модуль настроек (Settings)}}\label{md__r_e_a_d_m_e_autotoc_md12}
Класс {\ttfamily \doxylink{class_settings}{Settings}} предоставляет {\ttfamily Q\+\_\+\+PROPERTY} для доступа из QML. Параметры (хост, порт БД, сервер, режим, пути к логам и отчётам) сохраняются через {\ttfamily QSettings} в системном реестре/\+конфигурационном файле. Методы {\ttfamily load()} и {\ttfamily save()} вызываются при запуске и при закрытии диалога настроек.\hypertarget{md__r_e_a_d_m_e_autotoc_md13}{}\doxysubsection{\texorpdfstring{Пользовательский интерфейс (QML)}{Пользовательский интерфейс (QML)}}\label{md__r_e_a_d_m_e_autotoc_md13}

\begin{DoxyItemize}
\item {\bfseries{Файл настроек}} – хранится через {\ttfamily QSettings} (реестр Windows или конфигурационный файл). Вручную не редактировать.
\item {\bfseries{Лог-\/файл}} – по умолчанию {\ttfamily logs.\+txt} в рабочей папке. При ошибке записи используется системная папка приложения.
\item {\bfseries{Папка отчётов}} – задаётся в диалоге настроек, используется для сохранения CSV-\/файлов.
\end{DoxyItemize}

{\bfseries{Требования:\+}}
\begin{DoxyItemize}
\item Qt 6.\+8 или новее (Core, Quick, Sql, Network)

\end{DoxyItemize}

{\bfseries{Шаги сборки:\+}}


\begin{DoxyCode}{0}
\DoxyCodeLine{git\ clone\ <url>}
\DoxyCodeLine{cd\ RestN}
\DoxyCodeLine{mkdir\ build\ \&\&\ cd\ build}
\DoxyCodeLine{cmake\ ..}
\DoxyCodeLine{cmake\ -\/-\/build\ .\ -\/-\/config\ Release}

\end{DoxyCode}


{\bfseries{Запуск на Android:\+}} выберите в Qt Creator комплект Android, соберите APK, установите на планшет. На серверном устройстве должно быть установлено приложение (режим сервера) или отдельный Postgre\+SQL.
