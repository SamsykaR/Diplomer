# Пользовательская документация для программы автоматизации заказов ресторана (RestN)

## Обзор проекта

**Название:** RestaurantOrders  
**Тип:** кроссплатформенное приложение для планшетов (Android / Windows / Linux) на Qt 6  
**Назначение:** автоматизация приёма заказов, управления складом и формирования отчётности в ресторане.

**Ключевые возможности:**
- Авторизация сотрудников (официант, кладовщик, администратор)
- Оформление заказов с автоматическим списанием ингредиентов
- Возврат продуктов при удалении позиции из заказа
- Управление складскими остатками (добавление, убавка, установка)
- Отчёты по выручке, популярности блюд и дефицитным продуктам
- Сохранение отчётов в CSV
- Ролевое разграничение доступа
- Синхронизация данных по локальной сети (клиент-сервер)

## Описание ключевых модулей (для разработчиков)

### `DBWorker` (серверный режим)
- **Назначение:** прямое взаимодействие с PostgreSQL.
- **Основные методы:**
  - `checkUser(login, password)` – аутентификация с SHA3-256 + соль.
  - `getFoods()` – список блюд с доступностью и составом.
  - `checkAndConsume(foodId, quantity)` – списание ингредиентов в транзакции.
  - `returnIngredients(foodId, quantity)` – возврат продуктов.
  - `getReportRevenue()`, `getRevenueByDay()`, `getFoodPopularity()`, `getReportLowStock()` – формирование отчётов.
  - `saveReportToCSV()` – сохранение отчёта в CSV.

### `TcpClient` (клиентский режим)
- **Назначение:** удалённое взаимодействие с сервером через TCP.
- Дублирует интерфейс `DBWorker`, отправляя JSON-запросы.

### `TcpServer` + `ClientConnection`
- **Назначение:** приём и обработка клиентских запросов, вызов методов `DBWorker`.

### `Settings`
- **Назначение:** хранение параметров подключения, режима работы, путей к логам и папке отчётов. Использует `QSettings`.

### `Logs`
- **Назначение:** запись сообщений (INF, WARN, ERR) в файл и консоль.

### Пользовательский интерфейс (QML)
- **LoginPage.qml** – экран входа.
- **Main.qml** – главное окно с выдвижным меню и стеком навигации.
- **OrderPage.qml** – оформление заказа.
- **StockPage.qml** – управление складом.
- **ReportsPage.qml** – отчёты.

## Руководство пользователя

### Начальный запуск

1. Установите приложение на планшет (Android APK или запуск под Windows).
2. При первом запуске откроется окно авторизации.
3. Если приложение запускается в **клиентском режиме**, предварительно убедитесь, что **сервер** (другое устройство с той же программой в режиме сервера) работает и доступен по сети.
4. При необходимости настройте подключение через кнопку «Настройки» (см. раздел «Настройки»).

### Окно авторизации

- Поле **«Логин»** – введите имя пользователя.
- Поле **«Пароль»** – введите пароль (символы скрываются).
- Кнопка **«Войти»** – выполняет проверку данных. При успехе открывается главное окно с меню, соответствующим роли (официант, кладовщик, администратор).
- Кнопка **«Настройки»** – открывает диалог настройки подключения к серверу и базе данных.

### Диалог настроек

Доступен из окна авторизации или из выдвижного меню.

**Группа «База данных» (видна только в серверном режиме):**
- **Хост, порт, имя БД, пользователь, пароль** – параметры подключения к PostgreSQL.
- Кнопка **«Проверить подключение»** – тестирует соединение.

**Группа «Сеть»:**
- **IP сервера** – адрес устройства, на котором запущен сервер (для клиентского режима).
- **Порт сервера** (по умолчанию 12345).
- **Флажок «Режим сервера»** – если включён, приложение будет принимать клиентские подключения и работать напрямую с БД.
- Кнопка **«Проверить сервер»** – проверяет доступность сервера по указанному IP и порту.
- **«Показать IP устройства»** – отображает локальные IP-адреса текущего устройства (для настройки клиентов).

**Группа «Прочее»:**
- **Путь к логам** – имя файла для записи событий (по умолчанию `logs.txt`).
- **Папка для отчётов** – каталог, куда будут сохраняться CSV-файлы.

**Сохранение:** кнопки **OK** (сохранить и закрыть) и **Отмена** (отменить изменения).

### Главное окно и выдвижное меню

После авторизации отображается главное окно:

- **Верхняя панель:**
  - Кнопка **☰** – открывает выдвижное меню (Drawer).
  - Название «Ресторан».
  - Кнопка **⟳** – принудительная синхронизация данных (обновляет списки блюд, продуктов, отчётов).

**Выдвижное меню:**
- Вверху отображается ФИО текущего пользователя.
- Пункты зависят от роли:
  - **Официант** – «Оформление заказа».
  - **Кладовщик** – «Склад».
  - **Администратор** – «Оформление заказа», «Склад», «Отчёты».
- Кнопка **«Настройки»** – открывает диалог настроек.
- Кнопка **«Выйти»** – завершает сеанс и возвращает к окну авторизации.
- Кнопка **«Закрыть приложение»** – полностью выходит из программы.

Для закрытия меню коснитесь области вне его или нажмите кнопку **☰** повторно.

### Оформление заказа (страница для официанта)

Страница разделена на две колонки.

**Левая колонка – меню:**
- **Фильтр по категории** – выпадающий список (Супы, Горячее, Салаты, Напитки, Все).
- **Флажок «Только доступные»** – показывает только блюда, доступные по остаткам.
- Список блюд: название, цена, категория, описание, состав. Недоступные блюда подписаны красным «Нет в наличии».
- Чтобы выбрать блюдо – коснитесь строки (она подсветится).

**Правая колонка – оформление:**
- **«Текущий заказ»** – список добавленных блюд с количеством.
- Поле **«Столик №»** – номер столика (1–50).
- Поле **«Кол-во»** – количество порций (1–10).
- Поле **«Комментарий»** – особые пожелания (например, «без лука»).
- Кнопка **«Добавить»** – добавляет выбранное блюдо в заказ. Перед добавлением проверяется наличие ингредиентов; если после списания остатки станут ниже порога – появится предупреждение (добавление разрешается). Если ингредиентов не хватает – сообщение об ошибке.
- Кнопка **«Удалить»** – убирает выбранное блюдо из корзины и возвращает ингредиенты на склад.
- Кнопка **«Оформить заказ»** – фиксирует заказ в системе (списание ингредиентов, создание записи в БД). После успешного оформления корзина очищается, и список блюд обновляется.

### Управление складом (страница для кладовщика)

Отображает полный список продуктов.

- **Список продуктов:** название и текущее количество в единицах измерения (кг, л, шт.). Продукты с остатком ниже минимального порога подсвечены розовым фоном.
- Выбор продукта – коснитесь строки.
- **Поле «Количество»** – ввод целого числа (0–10000).
- Кнопка **«Добавить»** – увеличивает остаток выбранного продукта на введённое значение.
- Кнопка **«Убавить»** – уменьшает остаток (если результат неотрицательный).
- Кнопка **«Установить»** – устанавливает точное значение остатка.

Все изменения немедленно отправляются на сервер и синхронизируются с другими клиентами при нажатии кнопки синхронизации.

### Отчёты (страница для администратора)

Позволяет формировать три типа отчётов.

**Выбор типа отчёта:** выпадающий список вверху страницы.

**1. Выручка**
- Фильтры: дата «с», дата «по» (формат ГГГГ-ММ-ДД), блюдо, столик, официант.
- Кнопка **«Сформировать»** – отображает общую выручку и разбивку по дням.

**2. Популярность блюд**
- Фильтры: дата «с» и «по».
- Результат – список блюд с общим количеством проданных порций, отсортированный по убыванию.

**3. Малые остатки**
- Без фильтров. Отображаются продукты, у которых текущее количество ниже минимального порога.

**Сохранение отчёта:** после формирования нажмите **«Сохранить отчёт»**, укажите имя файла в диалоге. Файл будет создан в папке, указанной в настройках (по умолчанию `reports` в текущей директории). Формат – CSV (разделитель запятая, кодировка UTF-8).

## Настройки и системные файлы

- **Файл настроек:** сохраняется через `QSettings` в системном реестре (Windows) или конфигурационном файле (Linux/macOS). Вручную изменять не рекомендуется.
- **Лог-файл:** по умолчанию `logs.txt` в рабочей папке. Если запись невозможна, используется системная папка приложения (`AppLocalDataLocation`).
- **Папка отчётов:** задаётся в диалоге настроек, используется для сохранения CSV-файлов.

## Сборка проекта (для разработчиков)

**Требования:**
- Qt 6.8 или новее (Core, Quick, Sql, Network)
- CMake ≥ 3.16
- PostgreSQL (серверный режим) или возможность запуска клиентской части

**Шаги сборки:**
1. Клонировать репозиторий:
   ```bash
   git clone <url>
   cd RestaurantOrders
   ```
2. Создать папку сборки:
   ```bash
   mkdir build && cd build
   ```
3. Сконфигурировать CMake:
   ```bash
   cmake ..
   ```
4. Собрать проект:
   ```bash
   cmake --build . --config Release
   ```
5. Исполняемый файл `appapp` появится в папке сборки.

**Запуск на планшете Android:**
- Соберите APK через Qt Creator (выберите комплект Android).
- Установите на планшет.
- На устройстве-сервере также должно быть установлено приложение (режим сервера) или отдельный сервер PostgreSQL.
