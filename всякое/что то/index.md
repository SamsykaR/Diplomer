### Пользовательская документация для проекта «База данных метеоритов» (Doxygen)

#### 1. Обзор проекта

**Название:** База данных метеоритов (MeteoriteDB)  
**Тип:** настольное приложение на Qt6  
**Назначение:** учёт и визуализация данных о метеоритах с возможностью редактирования, фильтрации и экспорта.  

**Ключевые возможности:**
* управление записями о метеоритах (добавление, редактирование, удаление);
* визуализация данных в табличном виде и на графиках;
* импорт данных из CSV/JSON;
* экспорт данных в CSV;
* гибкая настройка базы данных и логов;
* логирование операций и ошибок.

#### 3. Описание классов

##### 3.1. `MainWindow`
**Файл:** `include/MainWindow.h`  
**Родитель:** `QMainWindow`  
**Описание:** главное окно приложения с вкладками. Сохраняет и загружает позицию и размер окна через `QSettings`.  

**Методы:**
* `MainWindow(QWidget *parent = nullptr)` — конструктор, инициализирует компоненты;
* `~MainWindow()` — деструктор, сохраняет настройки перед закрытием;
* `setupUI()` — создаёт и настраивает интерфейс (вкладки, макет);
* `loadSettings()` — загружает настройки окна (позиция, размер) из `settings.ini`;
* `saveSettings()` — сохраняет настройки окна в `settings.ini`.

**Связи:** использует `DBManager`, `Logger` и все вкладки (`TabInfo`, `TabEdit` и т. д.).

##### 3.2. `DBManager`
**Файл:** `include/DBManager.h`  
**Родитель:** `QObject`  
**Описание:** класс для работы с SQLite. Обеспечивает подключение к БД, создание таблиц, выполнение CRUD‑операций и обработку ошибок.  

**Основные методы:**
* `bool connectToDatabase(const QString &databasePath)` — подключает к БД по указанному пути;
* `bool createTables()` — создаёт таблицу `meteorites`, если её нет;
* `bool deleteDatabase()` — удаляет файл БД;
* `bool isTableExists() const` — проверяет существование таблицы `meteorites`;
* `QSqlDatabase database() const` — возвращает объект подключения к БД.

**Таблица `meteorites`:**
* `id` (`INTEGER`, PRIMARY KEY, AUTOINCREMENT) — уникальный идентификатор;
* `name` (`TEXT`) — название метеорита;
* `location` (`TEXT`) — место падения;
* `type` (`TEXT`) — тип метеорита;
* `mass` (`REAL`) — масса в кг;
* `discovery_year` (`INTEGER`) — год обнаружения;
* `chemical_composition` (`TEXT`) — химический состав;
* `scientific_value` (`TEXT`) — научное значение.

##### 3.3. `Logger`
**Файл:** `include/Logger.h`  
**Родитель:** `QObject`  
**Описание:** класс логирования. Записывает сообщения с временными метками, очищает и удаляет лог‑файл.  

**Основные методы:**
* `void log(const QString &message, const QString &level = "INFO")` — записывает сообщение в лог;
* `bool clearLog()` — очищает содержимое лог‑файла;
* `bool deleteLog()` — удаляет лог‑файл;
* `qint64 getLogSize() const` — возвращает размер лога в байтах;
* `QDateTime getLastModified() const` — возвращает дату последнего изменения лога.

**Формат лога:**
```
[LEVEL] yyyy-MM-dd hh:mm:ss: сообщение
```
Пример:
```
[ERROR] 2024-05-20 14:30:15: Ошибка подключения к БД: resources/database.db
```

##### 3.4. `TabSettings`
**Файл:** `include/TabSettings.h`  
**Родитель:** `QWidget`  
**Описание:** вкладка настроек приложения. Управляет путём к БД, созданием/удалением БД и таблиц, мониторингом и управлением логами.  

**Элементы интерфейса:**
* поле ввода пути к БД (`QLineEdit`);
* кнопка «Обзор» (`QPushButton`) — открывает диалог выбора файла;
* кнопки «Создать БД и таблицу» и «Удалить БД и таблицу»;
* информация о логе (`QLabel`) — размер и дата изменения;
* кнопки «Очистить лог» и «Удалить лог»;
* статус‑индикаторы (`QLabel`): «Статус БД» (подключено/не подключено), «Статус лога» (доступен/недоступен);
* кнопка «Сохранить настройки».

**Основные методы:**
* `onBrowseDatabasePath()` — открывает диалог выбора файла БД;
* `onCreateDatabase()` — создаёт БД и таблицу `meteorites`;
* `onDeleteDatabase()` — запрашивает подтверждение и удаляет БД;
* `onClearLog()` — очищает лог;
* `onDeleteLog()` — удаляет лог;
* `onSaveSettings()` — сохраняет путь к БД в `settings.ini`;
* `updateStatusIndicators()` — обновляет статус‑индикаторы и информацию о логе.

##### 3.5. Другие вкладки
* **TabInfo** (`TabInfo.h/cpp`) — отображает и редактирует информацию о студенте (ФИО, группа, программа) из `settings.ini`.
* **TabEdit** (`TabEdit.h/cpp`) — форма ввода/редактирования записей о метеоритах. Взаимодействует с `DBManager` для сохранения данных. Поддерживает drag‑and‑drop для импорта CSV/JSON.
* **TabTable** (`TabTable.h/cpp`) — таблица (`QTableView`) с данными из БД. Реализует фильтрацию, сортировку и экспорт в CSV. Привязана к `QSqlTableModel`.
* **TabGraphs** (`TabGraphs.h/cpp`) — визуализирует данные: карта мест падений и гистограмма распределения по массе. Позволяет сохранять диаграммы в изображения.

#### 4. Использование приложения

##### 4.1. Запуск
1. Соберите проект с помощью CMake:
   ```bash
   mkdir build && cd build
   cmake ..
   make
   ./MeteoriteDB
   ```
2. Приложение запустится с главным окном и вкладкой «Информация».

##### 4.2. Вкладки
* **Информация:** заполните и сохраните данные о студенте в `settings.ini`. Нажмите «О программе» для просмотра описания функционала.
* **Редактирование:** заполните поля формы и используйте кнопки:
    * «Добавить» — добавляет новую запись в БД;
    * «Сохранить» — обновляет существующую запись;
    * «Удалить» — удаляет выбранную запись;
    * «Очистить форму» — сбрасывает поля ввода.
    Поддерживается drag‑and‑drop CSV/JSON для импорта данных.
* **Таблица данных:** отображает содержимое БД в виде таблицы. Используйте поле поиска над таблицей для фильтрации по столбцам. Сортируйте данные кликом на заголовок столбца. Нажмите кнопку «Экспорт в CSV» для сохранения данных.
* **Диаграммы:** визуализирует данные на карте и гистограмме. Нажмите «Обновить данные» для пересчёта графиков. Сохраните диаграмму в изображение кнопкой «Сохранить как изображение».
* **Настройки:**
    * установите путь к БД через поле ввода или кнопку «Обзор»;
    * создайте БД кнопкой «Создать БД и таблицу»;
    * удалите БД кнопкой «Удалить БД и таблицу» (с подтверждением);
    * очистите или удалите лог кнопками «Очистить лог»/«Удалить лог»;
    * сохраните настройки кнопкой «Сохранить настройки»;
    * отслеживайте статус подключения к БД и доступности лога по индикаторам.

##### 4.3. Настройки и логи
* **Файл настроек:** `resources/settings.ini` (хранит путь к БД, позицию окна и т. д.).
* **База данных:** `resources/database.db` (SQLite).
* **Лог‑файл:** `resources/log.txt` (текстовый файл с записями операций и ошибок).

#### 5. Сборка проекта

**Требования:**
* Qt6 (Core, Widgets, Sql, Graphs);
* CMake ≥ 3.22;
* компилятор с поддержкой C++20.

**Шаги сборки:**

1. **Клонирование репозитория:**
```bash
git clone <url-репозитория>
cd MeteoriteDB
```

2. **Создание директории сборки:**
```bash
mkdir build
cd build
```

3. **Конфигурация проекта через CMake:**
```bash
cmake ..
```
Или с указанием компилятора:
```bash
cmake -DCMAKE_CXX_COMPILER=g++ ..
```

4. **Компиляция проекта:**
```bash
make
```
Для ускорения сборки можно использовать параллельную компиляцию:
```bash
make -j$(nproc)
```

5. **Запуск приложения:**
```bash
./MeteoriteDB
```

**Возможные проблемы при сборке и их решения:**

* **Ошибка: «Qt6 не найден»** — установите Qt6 через пакетный менеджер:
    * Ubuntu/Debian: `sudo apt install qt6-base-dev qt6-tools-dev`
    * Fedora: `sudo dnf install qt6-qtbase-devel qt6-qtsvg-devel`
    * Windows: установите Qt через Qt Online Installer.
* **Ошибка компиляции C++20** — убедитесь, что компилятор поддерживает C++20:
    * GCC ≥ 10;
    * Clang ≥ 12;
    * MSVC ≥ 2019 (с флагом `/std:c++20`).
* **Отсутствие библиотек Qt6::Graphs** — установите компонент Graphs:
    * В Qt Online Installer выберите модуль `Qt Graphs`.

### 6. Конфигурация и настройки

#### 6.1. Файл `settings.ini`

**Расположение:** `resources/settings.ini`.  
**Формат:** INI (ключ=значение).  

**Основные секции и параметры:**

* **[database]**
    * `path` — путь к файлу БД (по умолчанию: `resources/database.db`).
* **[window]**
    * `position` — позиция главного окна (формат: `x,y`);
    * `size` — размер главного окна (формат: `width,height`).
* **[student]**
    * `name` — ФИО студента;
    * `group` — номер группы;
    * `program` — название программы;
    * `version` — версия приложения.

**Пример содержимого:**
```ini
[database]
path=resources/database.db

[window]
position=100,100
size=1000,700

[student]
name=Иванов Иван Иванович
group=М-123
program=Прикладная информатика
version=1.0.0
```

#### 6.2. Файл базы данных

**Расположение:** `resources/database.db`.  
**Тип:** SQLite.  
**Таблица:** `meteorites` (создаётся автоматически при первом запуске).  

#### 6.3. Файл логов

**Расположение:** `resources/log.txt`.  
**Формат записей:**
```
[LEVEL] yyyy-MM-dd hh:mm:ss: сообщение
```
**Уровни логирования:**
* `INFO` — информационные сообщения;
* `WARNING` — предупреждения;
* `ERROR` — ошибки.

### 7. Руководство пользователя

#### 7.1. Начальный запуск

1. Запустите приложение `MeteoriteDB`.
2. Перейдите на вкладку **«Настройки»**:
    * проверьте путь к БД (по умолчанию: `resources/database.db`);
    * нажмите **«Создать БД и таблицу»**, если файл БД отсутствует;
    * сохраните настройки кнопкой **«Сохранить настройки»**.
3. Перейдите на вкладку **«Информация»** и заполните данные о студенте.

#### 7.2. Работа с данными

**Добавление записи:**
1. Перейдите на вкладку **«Редактирование»**.
2. Заполните поля формы:
    * **name** — название метеорита;
    * **location** — место падения (координаты или описание);
    * **type** — тип (каменный, железный и т. д.);
    * **mass** — масса в кг;
    * **discovery_year** — год обнаружения;
    * **chemical_composition** — химический состав;
    * **scientific_value** — научное значение.
3. Нажмите кнопку **«Добавить»**.

**Редактирование записи:**
1. Выберите запись в таблице на вкладке **«Таблица данных»**.
2. Данные автоматически отобразятся в форме на вкладке **«Редактирование»**.
3. Внесите изменения и нажмите **«Сохранить»**.

**Удаление записи:**
1. Выберите запись в таблице или форме.
2. Нажмите **«Удалить»** на вкладке **«Редактирование»**.

**Импорт данных:**
1. Перетащите CSV/JSON‑файл в область формы на вкладке **«Редактирование»**.
2. Данные будут автоматически добавлены в БД.

#### 7.3. Визуализация данных

**На вкладке «Диаграммы»:**
* **Карта мест падений** — отображает географическое распределение метеоритов.
* **Гистограмма массы** — показывает распределение метеоритов по массе.
* Нажмите **«Обновить данные»** для пересчёта графиков.
* Сохраните диаграмму в изображение кнопкой **«Сохранить как изображение»**.

#### 7.4. Экспорт данных

**Экспорт в CSV:**
1. На вкладке **«Таблица данных»** нажмите кнопку **«Экспорт в CSV»**.
2. Выберите путь и имя файла.
3. Данные из таблицы будут сохранены в формате CSV.

#### 7.5. Управление настройками и логами

**Изменение пути к БД:**
1. В поле **«Путь к БД»** на вкладке **«Настройки»** введите новый путь или выберите файл кнопкой **«Обзор»**.
2. Нажмите **«Сохранить настройки»**.

**Очистка логов:**
1. Нажмите **«Очистить лог»** на вкладке **«Настройки»**.
2. Содержимое `log.txt` будет удалено, сам файл сохранится.

**Удаление логов:**
1. Нажмите **«Удалить лог»**.
2. Подтвердите действие в диалоговом окне.
3. Файл `log.txt` будет удалён с диска.

### 8. Обработка ошибок

**Типичные ошибки и решения:**

* **«Не удалось подключиться к БД»**:
    * проверьте существование файла БД;
    * убедитесь, что путь к БД указан верно;
    * создайте БД кнопкой **«Создать БД и таблицу»**.
* **«Ошибка импорта CSV/JSON»**:
    * проверьте формат файла (должен соответствовать структуре таблицы `meteorites`);
    * убедитесь, что файл не повреждён.
* **«Нет прав доступа к файлу»**:
    * запустите приложение с правами администратора (если требуется);
    * измените права доступа к файлам в директории `resources/`.
* **«Ошибки логирования»**:
    * проверьте, что файл `log.txt` не заблокирован другим процессом;
    * очистите или удалите лог через вкладку **«Настройки»**.

### 9. Дополнительные рекомендации

* **Резервное копирование БД:** перед массовыми операциями (импорт, удаление) скопируйте файл `database.db` вручную.
* **Оптимизация производительности:** для больших объёмов данных используйте индексы в SQLite (можно добавить в `DBManager::createTables()`).
* **Расширение функционала:**
    * добавьте поддержку других форматов экспорта (JSON, Excel);
    * реализуйте поиск по всей БД на вкладке **«Таблица данных»**;
    * интегрируйте онлайн‑карты (например, OpenStreetMap) для визуализации мест падений.
