## 📋 **Техническое задание для разработки системы автоматизации ресторана**

### **1. Общая информация**
- **Тип приложения**: Кроссплатформенное мобильное приложение для планшетов + административная панель
- **Основная цель**: Автоматизация приёма заказов и управления складом в ресторане
- **Архитектура**: Клиент-серверная, TCP-соединения в локальной сети
- **Целевая ОС**: Android 9+ (для планшетов), кроссплатформенность через Qt

### **2. Функциональные требования**

#### **2.1. Модуль заказов**
```
- Создание нового заказа с привязкой к:
  • Номеру столика
  • Официанту (выбор из списка)
  • Времени создания
- Добавление блюд из меню в заказ:
  • Выбор из категорий (закуски, основные блюда, напитки и т.д.)
  • Указание количества
  • Особые пожелания (аллергены, исключения ингредиентов)
- Редактирование заказа (добавление/удаление позиций)
- Закрытие заказа с формированием чека
- Просмотр активных заказов в реальном времени
```

#### **2.2. Модуль меню**
```
- Централизованное управление меню через админ-панель:
  • Добавление/редактирование/удаление блюд
  • Поля: название, категория, цена, описание, состав
  • Фотография блюда (опционально)
- Автоматическая синхронизация меню на всех устройствах
- Возможность пометки блюда как "нет в наличии"
```

#### **2.3. Модуль склада**
```
- Учёт ингредиентов:
  • Название продукта
  • Единица измерения (кг, шт, л)
  • Текущий остаток
  • Минимальный запас (порог для оповещения)
- Автоматическое списание при оформлении заказа
- Ручное внесение изменений (поступление, списание)
- Отчёт о продуктах с низкими остатками
```

#### **2.4. Модуль отчётов**
```
- Фискальные чеки по каждому закрытому заказу (сохранение в БД)
- Отчёт о выручке за период с фильтрами:
  • По датам
  • По официантам
  • По столикам
  • По категориям блюд
- Аналитика среднего чека, количества гостей
```

#### **2.5. Пользовательские роли**
```
1. ОФИЦИАНТ:
   • Создание/редактирование заказов
   • Просмотр меню
   • Отправка заказа на кухню

2. КЛАДОВЩИК:
   • Управление остатками на складе
   • Внесение приходных накладных
   • Просмотр отчётов о низких остатках

3. АДМИНИСТРАТОР:
   • Управление меню
   • Просмотр всех отчётов
   • Управление пользователями
   • Настройка системы

4. АНАЛИТИК:
   • Просмотр аналитических отчётов
   • Экспорт данных
```

### **3. Технические требования**

#### **3.1. Технологический стек**
```
- Язык программирования: C++ (основная логика)
- Фреймворк UI: Qt 6.5 с Qt Quick (QML) для интерфейса
- База данных: SQLite (встроенная) для локального хранения
- Сетевое взаимодействие: TCP-сокеты для синхронизации
- Система сборки: CMake
- Целевые платформы: Android 9+, Windows (для админ-панели)
```

#### **3.2. Структура базы данных**
```sql
-- Основные таблицы:
1. waiters (id, name, login, password_hash)
2. tables (id, number, seats, status)
3. categories (id, name)
4. dishes (id, category_id, name, price, description, is_available)
5. ingredients (id, name, unit, current_quantity, min_quantity)
6. dish_ingredients (dish_id, ingredient_id, quantity)
7. orders (id, table_id, waiter_id, created_at, status)
8. order_items (id, order_id, dish_id, quantity, special_requests)
9. receipts (id, order_id, total_amount, created_at)
10. stock_transactions (id, ingredient_id, type, quantity, date, reason)
```

#### **3.3. Архитектура**
```
Клиентские приложения (планшеты) ↔ TCP-сервер ↔ База данных
                            ↑
                    Админ-панель (Windows)
```

### **4. Требования к интерфейсу**

#### **4.1. Общие принципы**
```
- Минималистичный дизайн
- Крупные элементы для тач-интерфейса
- Интуитивная навигация
- Минимум текста, максимум иконок
- Цветовая схема: нейтральные тона (серый, белый, синий акцент)
```

#### **4.2. Основные экраны**

**Экран входа:**
- Выбор роли (официант/кладовая/админ)
- Поле ввода логина/пароля

**Экран официанта:**
1. Вкладка "Активные заказы" - список столиков с заказами
2. Вкладка "Создать заказ" - выбор столика → выбор блюд из меню
3. Вкладка "Меню" - просмотр доступных блюд

**Экран кладовщика:**
1. "Остатки" - таблица продуктов с количеством
2. "Пополнение" - форма добавления продуктов
3. "Заявки" - список продуктов для заказа (ниже минимального)

**Экран администратора:**
1. "Управление меню" - CRUD для блюд и категорий
2. "Отчёты" - графики и таблицы продаж
3. "Персонал" - управление пользователями

### **5. Нефункциональные требования**

#### **5.1. Производительность**
```
- Время отклика интерфейса: < 200 мс
- Синхронизация данных между устройствами: < 2 секунд
- Одновременная работа: до 20 планшетов
- Хранение чеков: 3 года
```

#### **5.2. Безопасность**
```
- Авторизация по логину/паролю
- Хранение паролей в хэшированном виде (bcrypt)
- Разделение прав доступа по ролям
- Резервное копирование БД ежедневно
- Логирование всех действий в системе
```

#### **5.3. Надёжность**
```
- Автоматическое переподключение при потере сети
- Валидация данных при вводе
- Подтверждение критических действий (удаление, списание)
- Восстановление после сбоев
```
